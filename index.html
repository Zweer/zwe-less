Introduction

ZweLess is a LESS library that includes a lot of mixins and variables to help you styling a website. Nowadays there are a lot of different browsers, each with its set of prefixed properties and incompatibilities. ZweLess tries (and we can say succeded quite well) to think about that for you, saving you time to enjoy the styling process and enhance your creativity.

A keen eye could easily see that a lot of the code of ZweLess is taken from Preboot and from Less Hat. Of course all the credit for that goes to their creators, I've just put it in a single library, trying to make it better and more flexible.

Installation

Start using ZweLess is really easy: just download the project, copy the whole less directory into your website folder and include the zwe.less file: @import "zwe";.
Now you can start using all the mixins and variables provided by ZweLess

Variables

CSS doesn't have variables, but LESS does, and we try to take advantages of this feature to provide everything a designer wants.

Colors

Variables for colors could be too much, so we provide only grayscale: from #000 to #fff in steps of 10%: 
//Grayscale
@black-10: darken(#fff, 10%); // #e6e6e6
@black-20: darken(#fff, 20%); // #ccc
@black-30: darken(#fff, 30%); // #b3b3b3
@black-40: darken(#fff, 40%); // #999
@black-50: darken(#fff, 50%); // #808080
@black-60: darken(#fff, 60%); // #666
@black-70: darken(#fff, 70%); // #4d4d4d
@black-80: darken(#fff, 80%); // #333
@black-90: darken(#fff, 90%); // #191919
For example one of the above squares is colorized in this way: 
.black-10 {
  background: @black-10;
}
.black-10 {
  background: #e6e6e6;
}

Vendor Mixins

With all these browsers out there, writing prefixed properties can be really painfull. ZweLess allows you to write single-line mixins instead of multi-line properties, leaving you more time to design!

By default all these mixins use the default w3c parameter.

Backface Visibility (w3c)

Possible values are: visible (default) and hidden 
.backface-visibility(@visibility: visible) {
  -webkit-backface-visibility: @visibility;
     -moz-backface-visibility: @visibility;
      -ms-backface-visibility: @visibility;
          backface-visibility: @visibility;
}
Example: 
.test-backface-visibility {
  .backface-visibility(hidden);
}

.test-backface-visibility {
  -webkit-backface-visibility: hidden;
     -moz-backface-visibility: hidden;
      -ms-backface-visibility: hidden;
          backface-visibility: hidden;
}

Border Image (w3c)

.border-image(@image: none) {
  -webkit-border-image: @image;
       -o-border-image: @image;
          border-image: @image;
}
Example: 
.test-border-image {
  .border-image(url(border.png) 30 30 round);
}

.test-border-image {
  -webkit-border-image: url(border.png) 30 30 round;
       -o-border-image: url(border.png) 30 30 round;
          border-image: url(border.png) 30 30 round;
}

Border Radius (w3c)

Border Radius is supported by all major browsers. Sometimes, however, it doesn't display correctly the background. A background-clip: padding-box is applied to fix it. 
.border-radius(@radius: 0) {
    border-radius: @radius;
  background-clip: padding-box;
}
The same behaviour is applied to the single corners: 
.border-top-left-radius(@radius: 0) {
  border-top-left-radius: @radius;
         background-clip: padding-box;
}

.border-top-right-radius(@radius: 0) {
  border-top-right-radius: @radius;
          background-clip: padding-box;
}

.border-bottom-right-radius(@radius: 0) {
  border-bottom-right-radius: @radius;
             background-clip: padding-box;
}

.border-bottom-left-radius(@radius: 0) {
  border-bottom-left-radius: @radius;
            background-clip: padding-box;
}
Example: 
.test-border-radius {
  .border-radius(25px);
}

.test-border-radius {
    border-radius: 25px;
  background-clip: padding-box;
}
Moreover there are mixins to aggregate more corners: 
.border-top-radius(@radius: 0) {
  .border-top-left-radius(@radius);
  .border-top-right-radius(@radius);
}

.border-right-radius(@radius: 0) {
  .border-top-right-radius(@radius);
  .border-bottom-right-radius(@radius);
}

.border-bottom-radius(@radius: 0) {
  .border-bottom-left-radius(@radius);
  .border-bottom-right-radius(@radius);
}

.border-left-radius(@radius: 0) {
  .border-top-left-radius(@radius);
  .border-bottom-left-radius(@radius);
}
Example: 
.test-border-top-radius {
  .border-top-radius(10px);
}

.test-border-top-radius {
  border-top-right-radius: 10px;
   border-top-left-radius: 10px;
          background-clip: padding-box;
}
As you can see, LESS automatically removes the duplicated background-clip

Box Shadow (w3c)

.box-shadow(@shadow: none) {

.test-box-shadow {

  -webkit-box-shadow: 0 1px 10px #000000, inset 0 -2px 5px #ff0000;

}

Opacity (w3c)

The opacity property doesn't have vendor prefixes, but it isn't compatible with IE8 and earlier versions. To extend this compatibility the alpha filter is used. 
.opacity(@opacity: 1) {
  opacity: @opacity;

  @opacity-ie: @opacity * 100;
   filter: ~"alpha(opacity=@{opacity-ie})"; // IE8
}
Example: 
.test-opacity {
  .opacity(.5);
}

.test-opacity {
  opacity: 0.5;
   filter: alpha(opacity=50);
}

Transform (w3c)

Since IE and Opera do not support 3D transformations, we have divided 3D from 2D prefixes: 
.transform3d(@transform: none) {
  -webkit-transform: @transform; // Chrome && Safari && iOS && Android
     -moz-transform: @transform; // Firefox < 16
          transform: @transform;
}

.transform(@transform: none) {
  -ms-transform: @transform; // IE 9
   -o-transform: @transform; // Opera < 12.1
  .transform3d(@transform);
}
Example: 
.test-transform {
  .transform(rotate(7deg));
}

.test-transform {
  -webkit-transform: rotate(7deg);
     -moz-transform: rotate(7deg);
      -ms-transform: rotate(7deg);
       -o-transform: rotate(7deg);
          transform: rotate(7deg);
}

.test-transform3d {
  .transform3d(rotate(7deg));
}

.test-transform3d {
  -webkit-transform: rotate(7deg);
     -moz-transform: rotate(7deg);
          transform: rotate(7deg);
}

Utilities

How many times have you been in the situation to add the "clearfix" class in a project? Or jumping to Font Squirrel to remember how font-face works?

These mixins try to provide you all these functions you are going to use at least once per project.

Center Block

Centers something in the page/container. It must be displayed as a block, so the display: block property is automatically added. 
.center-block {
      display: block;

   margin-left: auto;
  margin-right: auto;
}
As you can see, this mixin is more a class than a real mixin, so that you can use it also in the html of your project.

Clearfix (source)

Adding an empty div to clear the floating elements is always really bad practice. With this class we put it in the container block to clear all the elements inside. 
.clearfix {
  *zoom: 1; // 3

  &:before,
  &:after {
    content: " "; // 1
    display: table; // 2
  }
  &:after {
    clear: both;
  }
}

Font Face

This mixin is more a shortcut than anything else: writing every time all the sources of a font is really boring and many times we do forget something: 
.font-face(@font-name, @font-file, @font-weight: normal, @font-style: normal) {
  @font-face {
    font-family: "@{font-name}";
    src: url("@{font-file}-webfont.eot");
    src: url("@{font-file}-webfont.eot?#iefix") format("embedded-opentype"),
         url("@{font-file}-webfont.woff") format("woff"),
         url("@{font-file}-webfont.ttf") format("truetype"),
         url("@{font-file}-webfont.svg#@{font-name}") format("svg");
    font-weight: @font-weight;
    font-style: @font-style;
  }
}

Size

This mixin stylizes the size of a block (display: block is usually needed). 
.size(@width, @height) {
   width: @width;
  height: @height;
}
If only one argument is passed, the mixin uses that @width also for the @height, like a square: 
.size(@square) {
  .size(@square, @square);
}

Text Truncate

When you want to truncate the text, you have to put other properties to make it happen without problems. This mixin put them automatically.

For it to happen you have to stylize the element as a block or an inline-block 
.text-truncate(@truncate-model: clip) {
     overflow: hidden;
  white-space: nowrap;
  .text-overflow(@truncate-model);
}
To make design easier there is also a class usable directly in the HTML, that truncates the text with the ellipsis property: 
.text-truncate {
  .text-truncate(ellipsis);
}
